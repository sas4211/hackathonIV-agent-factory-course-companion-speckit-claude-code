## Steps

1. Read `WORKLOG.md` to confirm the current phase and verify all its tasks are ✅.
2. If any tasks are not ✅, list them and stop — do not advance the phase until they're done.
3. If all tasks are complete:
   - Mark the current phase as `✅ Complete` in the Phase Overview table.
   - Mark the next phase as `🔄 In Progress`.
   - Append a Decisions Log entry with today's date noting the phase transition.
4. Print a handoff summary:
   - **Phase X delivered:** bullet list of what was built
   - **Phase X+1 first tasks:** the first 2–3 deliverables to tackle, with a brief note on approach

Do not start implementing Phase 2 work — this skill only handles the transition and briefing.
